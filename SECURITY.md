# Security Policy

## Supported versions

The project is still early, so the latest code on `main` and the current `0.1.x` line should be treated as supported.

## Reporting a vulnerability

Please do not open public GitHub issues for security problems.

Instead:

- If GitHub private vulnerability reporting is enabled for this repository, use that.
- Otherwise, contact the maintainers privately through GitHub or any security contact published for the project.

Please include as much detail as you can:

- affected version, branch, or commit
- operating system and environment
- reproduction steps or proof of concept
- impact assessment
- any ideas for mitigation

## Response expectations

- We aim to acknowledge new reports within 72 hours.
- We aim to provide an initial triage update within 7 days.
- We will work with reporters on a reasonable disclosure timeline once a fix is available.
